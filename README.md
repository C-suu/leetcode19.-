### （1）题目意思及生僻概念解释

**题目白话文解释：**
这道题的任务是操作一列单向连接的火车（链表）。已知火车的头节点，要求拆除倒数第 `n` 节车厢，并且把被拆车厢的前后两部分重新连接起来，最后返回这列火车新的头节点。
因为单向链表只能从前往后依次访问，无法直接从后往前数。如果不知道火车的总长度，要找到倒数第 `n` 节车厢就会比较困难。

**生僻概念解释：**

  * **哨兵节点 (Dummy Node)：** 也叫虚拟头节点。在操作链表时，如果刚好要删除的是原来的第一个节点（头节点），那么链表的起始位置就会发生变化，这通常需要写额外的代码来特殊处理。为了避免这种麻烦，可以在原有的头节点前面，人为地制造一个没有任何实际意义的“假节点”作为新的车头。这样一来，无论要删除的是哪一节车厢（哪怕是原来的第一节），它前面必定还有车厢，所有删除操作的逻辑就统一了。最后返回结果时，只需要返回这个“假节点”的下一节即可。
  * **双指针法 (Two Pointers)：** 这里使用的是一种叫做“前后指针”或者“快慢指针”的技巧。想象有两只游标（左指针和右指针），一开始都停在哨兵节点上。让右指针先单独往后走 `n` 步。此时左右指针之间就拉开了固定的距离 `n`。接着，让左右指针保持这个距离，同时往后走。当右指针走到最后一节车厢时，左指针正好停在**要拆除车厢的前一节**。找到前一节，就能轻松修改挂钩完成拆除了。

### （2）代码及详细注释

```python
class Solution:
    # 定义方法：接收链表头节点 head 和要删除的倒数第 n 个位置，返回新的头节点
    def removeNthFromEnd(self, head: Optional[ListNode], n: int) -> Optional[ListNode]:
        # 创建哨兵节点，并将其 next 指针指向原链表头节点 head。同时让 left, right, dummy 都指向这个哨兵节点
        left = right = dummy = ListNode(next=head)
        
        # 循环 n 次
        for _ in range(n):
            # 右指针向后移动一步，最终领先左指针 n 步
            right = right.next  
            
        # 当右指针的下一个节点存在时（说明右指针还没走到最后一个节点），继续循环
        while right.next:
            # 左指针向后移动一步
            left = left.next
            # 右指针向后移动一步（保持左右指针间距不变，一起平移）
            right = right.next  
            
        # 循环结束时，左指针正好在待删除节点的前一个位置。修改挂钩，跳过待删除节点
        left.next = left.next.next  
        
        # 返回哨兵节点的下一个节点，即真正的链表头节点
        return dummy.next
```

### （3）逐行代码详细中文解释

  * `class Solution:`
    定义包含解题方法的类。
  * `def removeNthFromEnd(self, head: Optional[ListNode], n: int) -> Optional[ListNode]:`
    定义核心函数，接收代表链表起点的 `head` 和整数 `n`，表示要删除倒数第 `n` 个节点。返回值是处理后的链表起点。
  * `left = right = dummy = ListNode(next=head)`
    这一行做了两件事。第一，利用 `ListNode(next=head)` 凭空制造了一个哨兵节点，并且把它的尾部挂钩直接连上了原有的 `head` 节点。第二，声明了三个变量 `left`（左指针）、`right`（右指针）和 `dummy`（记录哨兵节点位置的基准），并将它们同时指向刚刚造出来的这个哨兵节点。
  * `for _ in range(n):`
    开启一个执行 `n` 次的循环。`_` 表示在循环体内不需要用到循环计数器的具体数值。
  * `right = right.next`
    在上述循环内，每次把右指针向链表尾部挪动一个节点。循环结束后，右指针正好超前左指针 `n` 步。
  * `while right.next:`
    开启条件循环。检查右指针当前所在的节点是否是最后一个节点（即检查它的 `.next` 挂钩是不是空的）。只要挂钩后面还有节点，循环就继续。
  * `left = left.next`
    将左指针向后移动一个节点。
  * `right = right.next`
    将右指针也向后移动一个节点。这使得左右指针像一个固定长度的尺子一样，在链表上同步向右滑动。
  * `left.next = left.next.next`
    当上面的 `while` 循环停止时，右指针刚好踩在链表的最后一个节点上，而左指针正好踩在**必须被删除的那个节点的前一个节点**上。这行代码是删除操作的核心：把左指针所在节点的挂钩（`.next`），直接连到下下个节点上（`.next.next`）。这样中间的那个节点就从链表中脱落了。
  * `return dummy.next`
    运算全部完成。因为最初的 `head` 节点有可能在刚才的操作中被删除了，所以不能直接返回 `head`。必须返回哨兵节点 `dummy` 的下一个节点（`dummy.next`），这才是这列火车目前真正且安全的第一节车厢。

### （4）具体数值算例与变量变化过程

假设有一个链表：`1 -> 2 -> 3 -> 4 -> 5`，需要删除倒数第 `2` 个节点（也就是节点 `4`）。即 `n = 2`。

初始准备，加入哨兵节点（假设其值为 `0`）：
链表变为：`[0] -> 1 -> 2 -> 3 -> 4 -> 5`
此时 `left`, `right`, `dummy` 都在节点 `[0]` 上。

以下是程序执行的每一步状态表格：

| 执行阶段 | 操作指令 | 左指针 `left` 所指节点 | 右指针 `right` 所指节点 | 链表状态 / 指针距离 |
| :--- | :--- | :--- | :--- | :--- |
| **初始状态** | 创建哨兵节点 | `[0]` | `[0]` | `[0] -> 1 -> ...` <br>距离: 0 步 |
| **第一阶段: 拉开距离** | 循环 1 (`right` 走一步) | `[0]` | `1` | 距离: 1 步 |
| | 循环 2 (`right` 走一步) | `[0]` | `2` | 距离: 2 步 (完成 `n` 次循环) |
| **第二阶段: 同步平移** | 检查 `right.next` (指向3，不为空) | `1` | `3` | `left` 和 `right` 均走一步 |
| | 检查 `right.next` (指向4，不为空) | `2` | `4` | `left` 和 `right` 均走一步 |
| | 检查 `right.next` (指向5，不为空) | `3` | `5` | `left` 和 `right` 均走一步 |
| | 检查 `right.next` (为空，退出循环) | **`3`** | **`5`** | `right` 到达终点，`left` 到达目标前驱位置 |
| **第三阶段: 修改挂钩** | 执行 `left.next = left.next.next` | 仍在 `3` | 仍在 `5` | 节点 `3` 的 `.next` 原本指向 `4`，现在改指 `5`。<br>链表变为: `[0] -> 1 -> 2 -> 3 -> 5` |
| **返回结果** | 执行 `return dummy.next` | - | - | 哨兵 `[0]` 的下一个节点是 `1`。<br>最终返回链表: `1 -> 2 -> 3 -> 5` |
